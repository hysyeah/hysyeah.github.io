tcp-ip subnet

发表于 2020-11-09

IP地址:互联网中的设备为了能进行通信必须拥有一个唯一的网络地址，这个地址就是IP地址,就像电话号码一样，如果不唯一就无法进行通信。

ip地址为32位，并且分为不同的地址格式，共A,B,C,D,E类。


如果两个IP具有相同的网络号，刚说明这两个IP在同一个网段，也就是同一个子网。处于同一个子网的主机可以直接通信，而不用通过网关来通信。而子网掩码就是用于来标识两个IP是否位于同一网段。

对于不同的类型的IP,网络号也是不一样的。网络号的地址可以参考上图。

子网的计算:对IP地址和子网掩码进行求余
对于C类IP地址192.168.0.1，子网掩码:255.255.255.0，先转换为二进制，然后AND每一位

IP: 11000000.10101000.00000000.00000001

子网掩码:11111111.11111111.11111111.00000000

结果: 11000000.10101000.00000000.00000000

转换为十进制为:192.168.0.0,这就是IP的网络标识

CIDR表示法:比如IP:192.168.0.1,子网掩码:255.255.255.0,用CIDR的来表示则为 192.168.0.1/24。24表示子网掩码的前24位值为1,其它值都为0。

子网掩码和对应的CIDR可以参考如下的表格

子网计算器

子网的作用：
子网掩码将大的网络域，分为更小的子网。较小的网络会创建较小的广播，从而产生较少的广播流量。 此外，子网还通过将网络问题隔离到特定的存在来简化故障排除。

两个IP的通信过程:
假设A,B两个主机处于同一网段。那A和B是如何通信的呢?假从A中ping主机B,A会通过自己主机的子网掩码计算自己和B是否位于同一网段，如果位于同一网段,A会通过ARP获得B的MAC地址然后进行通信;如果不属于同一网段,跨子网通信需要默认网关的转发,而要和默认网关通信就需要获得其MAC地址,然后A将请求交给默认网关,网关接到请求后再转发给B。

Linux Namespace

发表于 2020-07-05 更新于 2020-07-11

Linux Namespace是容器的基石，可以说没有Linux Namespace就不存在现在的容器技术。容器最大的功能就是对资源进行隔离，容器正是通过Linux Namespace技术实现了各种资源的隔离。

Linux Namespace总共有6种类别的Namespace，分别为

Namespace system call flag kernal version
UTS Namespace CLONE_NEWUTS 2.6.19
Mount Namespace CLONE_NEWNS 2.4.19
IPC Namespace CLONE_NEWIPC 2.6.19
PID Namespace CLONE_NEWPID 2.6.24
Network Namespace CLONE_NEWNET 2.6.29
User Namespace CLONE_NEWUSER 3.8

1.UTS Namespace

UTS是UNIX Time Sharing的缩写。主要用来隔离hostname，在每个UTS Namespace下，每个Namespace都可以拥有自己的Namespace。

2.IPC Namespace

用于隔离System V IPC和POSIX message queues。

左边为宿主机，右边为新建的sh环境。新建的message queue在右边的隔离Namespace并不能看到。

3.PID Namespace

PID Namespace用于隔离进程ID

5.User Namespace

User Namespace主要是用于隔离用户的用户组ID。一个进程的User ID和Group ID在User Namespace内外是不同的。常用的场景是在宿主机以一个非root用户运行创建一个User Namespace`,然后在命名空间里面却映射成root用户。

6.Network Namespace

Network Namespace用于隔离网络设备，IP地址，端口，路由表，防火墙规则等网络栈的Namespace。每个容器独占一个Network Namespace，每个容器都能随意使用自己的端口而不会产生冲突。

类型和接口(Types and interfaces)

反射构建在类型系统之上。Go是静态类型语言，每个变量都有一个静态类型，在编译阶段确认。
